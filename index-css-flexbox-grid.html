CSS Grid — Guía Visual

El sistema de layout en dos dimensiones · con demo interactiva

¿Qué es CSS Grid?

Grid convierte un contenedor en una cuadrícula de filas y columnas. Los hijos se colocan dentro de esa cuadrícula. A diferencia de Flexbox (que trabaja en una sola dirección), Grid controla dos dimensiones a la vez.

.contenedor {
  display: grid;
  grid-template-columns: 200px 200px 200px; /* 3 columnas de 200px */
  gap: 16px; /* espacio entre celdas */
}
Propiedades esenciales

display: grid

Activa Grid en el contenedor. Los hijos directos se convierten en grid items automáticamente.

grid-template-columns

Define cuántas columnas hay y su tamaño. Cada valor es una columna: 1fr 1fr 1fr = tres columnas iguales.

grid-template-rows

Define el alto de las filas. Si no lo defines, las filas se adaptan al contenido automáticamente.

gap

Espacio entre todas las celdas. gap: 16px aplica igual en horizontal y vertical. gap: 24px 12px para controlar por separado.

repeat()

repeat(3, 1fr) es igual que 1fr 1fr 1fr. Evita repetir valores y hace el código más limpio.

grid-area

Asigna un nombre a un elemento para posicionarlo en un área nombrada del grid. Se usa junto con grid-template-areas.

Cómo se colocan los items automáticamente

Por defecto los items se colocan de izquierda a derecha, saltando a la siguiente fila cuando se llenan las columnas. No tienes que hacer nada especial — Grid lo gestiona solo.

La unidad fr — fracción del espacio disponible

fr significa "fracción". En vez de píxeles fijos, las columnas se reparten el espacio disponible proporcionalmente. Si el contenedor cambia de tamaño, las columnas se adaptan solas.

La combinación más usada en proyectos reales

grid-template-columns: repeat(3, 1fr);
/* 3 columnas que se reparten el espacio — siempre iguales */
grid-template-areas — el layout visual

Con grid-template-areas defines el layout escribiéndolo visualmente en el CSS. Lees el código y ves la página. Es la forma más legible de estructurar un layout completo.

.pagina {
  display: grid;
  grid-template-areas:
    "header header"
    "sidebar contenido"
    "footer footer";
  grid-template-columns: 140px 1fr;
  gap: 8px;
}

header { grid-area: header; }
aside { grid-area: sidebar; }
main { grid-area: contenido; }
footer { grid-area: footer; }

Grid vs Flexbox — cuándo usar cada uno

Esta es una de las preguntas más frecuentes en entrevistas junior. No hay una respuesta de "uno es mejor que el otro" — son herramientas complementarias.

Flexbox

▸Una sola dirección: fila O columna
▸Componentes: botones, nav, tarjetas, headers
▸Centrar elementos fácilmente
▸Distribuir espacio entre items en una línea
▸El tamaño de los items es flexible por defecto

Grid

▸Dos dimensiones: filas Y columnas a la vez
▸Layouts de página: header, sidebar, footer
▸Galerías, cuadrículas de tarjetas, dashboards
▸Posicionar items en ubicaciones exactas
▸Estructura definida desde el contenedor

La regla práctica para la entrevista

Piensas en una dirección (fila o columna) → Flexbox
Piensas en filas y columnas a la vez → Grid

En la práctica se usan juntos: Grid para la estructura general de la página, Flexbox para los componentes dentro de esa estructura.

Ejemplos concretos:

Usaría Flexbox para...

Menú de navegación · Botón con icono · Fila de badges · Centrar algo en pantalla · Lista horizontal

Usaría Grid para...

Layout completo de la página · Galería de imágenes · Dashboard de tarjetas · Cualquier cosa en cuadrícula
